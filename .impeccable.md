# Impeccable — design context

Formal **Design Context** (below) drives Impeccable craft/polish. **Codebase implementation notes** are technical guardrails inferred from the repo—confirm with product owner where they conflict with brand intent.

## Design Context

### Users

- **Who**: Prospective and existing clients of **Consumer Solutions**, a Portuguese **credit intermediary / mortgage broker**; emphasis in copy on **international investors** financing Portuguese real estate.
- **Context**: Mostly **marketing / information** browsing (services, certifications, contacts, legal pages), not a logged-in product UI. Users may compare firms, check regulatory standing, and request consultation (e.g. mailto CTA).
- **Jobs to be done**: Understand services and credibility → feel safe with regulation (Bank of Portugal messaging) → take next step (contact / explore services).
- **Emotions to aim for** *(confirm with you)*: Inferred **trust, clarity, calm authority**—not hype. Replace if you want a different emotional target.

### Brand Personality

- **Provisional (from copy + UI, not stakeholder-signed)**: Professional, **compliance-forward**, **globally legible** (English-led site + PT/FR hints, `next-intl`).
- **Your 3 words** *(please reply in chat)*: e.g. “warm and precise and bold”—these should replace or lock the line above.
- **Anti-pattern** *(optional)*: What must this site **not** feel like (e.g. crypto-fintech neon, playful consumer app, generic bank template)?

### Aesthetic Direction

- **Current implementation**: **Light-first** chrome (white header/footer, black/gray type); **green** as primary CTA (`#368867`, `green-800` outlines); **Oswald** as global body font; hero **full-bleed video** + large display type; dark **near-black band** for secondary CTA (`hsl(0 0 9)`); responsive **`portrait:`** padding shifts.
- **Theme decision** *(please confirm)*: Should we treat the product as **light-only**, **dark-first**, or **both** (system `prefers-color-scheme` / toggle)? Code mixes `dark:` and `prefers-color-scheme` in CSS—need single intent for future work.
- **References** *(optional)*: Any sites/apps whose **tone or layout** you want to echo (and what specifically: typography, spacing, motion)?

### Design Principles

1. **Trust before flair**: Regulation and plainspoken copy outweigh decorative trends; every visual choice should support credibility.
2. **One accent discipline**: Green stays **sparse** (CTAs, key links)—avoid diluting it across every surface.
3. **Hierarchy over noise**: Few type sizes, strong contrast (hero already uses large display scale); keep long text within readable measure (~65–75ch) on new sections.
4. **i18n by default**: New strings go through **next-intl** / `useTranslations` where siblings do; avoid orphan hard-coded English on localized routes.
5. **Accessible interaction**: Visible **focus** states and contrast checks when changing colors; respect **reduced motion** for any new animation.
